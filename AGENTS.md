# Agent / AI Assistant Guide

This file applies to AI coding assistants (Claude Code, Cursor, Codex,
Copilot, etc.) helping a contributor add or modify a waste collection
source.

## Required reading before you suggest changes

1. **[doc/contributing_source.md](doc/contributing_source.md)** —
   adding a new dedicated source.
2. **[doc/contributing_ics.md](doc/contributing_ics.md)** — adding an
   ICS-based source via the existing YAML provider definitions (try this
   path first if the provider exposes a public ICS feed).
3. **[custom_components/waste_collection_schedule/i18n/README.md](custom_components/waste_collection_schedule/i18n/README.md)** —
   how parameter labels, descriptions, and error messages get
   translated. This catches the most common AI-introduced regression
   (see "Common pitfalls" below).

## Common pitfalls

### ❌ Don't add `PARAM_TRANSLATIONS` or `PARAM_DESCRIPTIONS` dicts

These dicts are no longer recognised. `tests/test_i18n.py` will block
the PR if either appears in a source module.

Argument labels and descriptions live in YAML next to the source:

```
custom_components/waste_collection_schedule/waste_collection_schedule/source/
├── my_source.py
└── my_source.i18n/                 only when needed
    ├── en.yaml
    └── de.yaml                     for non-English sources
```

For common parameter names (`address`, `city`, `street`, `postcode`,
`house_number`, `uprn`, …) **don't create any YAML at all** — labels
resolve from the shared registry at
`custom_components/waste_collection_schedule/i18n/shared/<lang>.yaml`.

### ❌ Don't hand-write English error messages

Use the structured exceptions from `waste_collection_schedule.exceptions`
(`SourceArgumentNotFound`, `SourceArgumentRequired`, etc.) so messages
render in the user's Home Assistant locale.

### ❌ Don't manually edit generated files

Auto-generated by `update_docu_links.py`:
- `README.md`, `info.md`
- `custom_components/waste_collection_schedule/sources.json`
- `custom_components/waste_collection_schedule/source_metadata.json`
- `custom_components/waste_collection_schedule/translations/*.json`
- `doc/ics/*.md` (regenerated from `doc/ics/yaml/*.yaml`)

`doc/source/*.md` is **not** auto-generated and must be created
manually for each new source.

### ❌ Don't hardcode dates, schedules, or test fixtures into source code

Sources must fetch live data from the provider (API, ICS, scraped
website). Hardcoded data is rejected during review.

## Required workflow when modifying a source

1. Edit the `.py` source module (and `<id>.i18n/<lang>.yaml` overrides
   if needed).
2. Run from the repo root: `python update_docu_links.py`. This
   regenerates `README.md`, `info.md`, `sources.json`,
   `source_metadata.json`, and `translations/*.json`.
3. Run tests: `python -m pytest tests/`. The `tests/test_i18n.py`
   suite enforces the rules above.
4. For new sources: create a corresponding `doc/source/<id>.md`
   manually.

## Project context

This is a Home Assistant custom component (HACS) that retrieves waste /
bin collection schedules from ~600 service providers worldwide.
PRs go upstream to `mampfes/hacs_waste_collection_schedule`.
